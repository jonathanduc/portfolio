Open Data Dashboard — OMS (GHO)

Projet en binôme — M2 (Open Data & Web des données) — Jonathan Duckes & Audric Girondin

Application web Streamlit d’exploration et d’analyse de données de santé publique issues de l’API GHO de l’OMS. L’app récupère dynamiquement les indicateurs, propose des analyses descriptives & visualisations, et intègre un module Machine Learning : régression linéaire (prévision temporelle) et clustering KMeans (profilage pays via PCA).

  • Voir l’application
  • Voir sur GitHub

Objectifs

  • Rendre des indicateurs OMS accessibles via un dashboard interactif.
  • Permettre une analyse temporelle (tendances & prévisions) et un profilage de pays.
  • Proposer des visualisations géographiques pour comparer les régions.

Fonctionnalités principales

  • Analyse de données : sélection d’un indicateur “Health*” → stats descriptives, graphes (barres/lignes), filtres pays/années.
  • ML — Régression : droite de tendance + prédictions, coefficient/intercept/score R².
  • ML — Clustering : normalisation, PCA (2D) et KMeans (sélection via méthode du coude).
  • Cartographie : choroplèthe (Plotly), carte GeoPandas/Folium pour l’agrégation pays.

Stack & données

  • Python — Streamlit, Pandas, NumPy, scikit-learn (LinearRegression, KMeans, PCA)
  • Visualisation — Plotly, GeoPandas, Folium
  • Source — OMS / GHO API (indicateurs, séries par pays & année)

Défis & améliorations

  • Données manquantes : filtrage & imputation.
  • Latence API : limitation d’indicateurs / cache.
  • Évolutions : + d’indicateurs, modèles ML avancés, actualisation planifiée.
