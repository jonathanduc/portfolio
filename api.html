Recipes & Nutrition — Data Integration API

Projet en binôme — Node.js / Express + Spoonacular & Open Food Facts — Swagger - Audric Girondin & Jonathan Duckes

Application web permettant de découvrir des recettes et d’afficher, pour chaque ingrédient, des informations nutritionnelles (dont NutriScores) en connectant deux APIs : Spoonacular (recettes) et Open Food Facts (nutrition). Le backend Node.js / Express sert de couche d’intégration : il agrège, normalise et expose les données via une API unique, documentée avec Swagger.

  • Voir sur GitHub

Objectifs

  • Unifier des sources hétérogènes (recettes + nutrition) derrière une API REST unique.
  • Fournir une UI web simple pour filtrer type de recette, cuisine & nombre de résultats.
  • Documenter et tester l’API via Swagger UI (/api-docs).

Architecture & Méthodologie

  • Backend (Express) : routes REST, validation d’inputs, normalisation de schémas.
  • APIs externes : proxification des appels Spoonacular & Open Food Facts.
  • Config : variables d’environnement (.env) pour clés & ports.
  • Frontend : HTML/CSS/JS + lite-server pour le dev (rechargement auto).
  • Docs : Swagger (OpenAPI) exposé sur /api-docs.

Endpoints (exemples)

  • GET /recipes?type={dessert|main}&cuisine={french|...}&limit=10 — liste de recettes (titre, image, ingrédients)
  • GET /nutrition?ingredient=tomato — infos nutritionnelles & NutriScore par ingrédient

Stack & Outils

  • Backend : Node.js, Express, Swagger (OpenAPI)
  • Frontend : HTML, CSS, JavaScript, lite-server
  • Intégrations : Spoonacular API, Open Food Facts

Résultats & Points forts

  • Parcours simple : filtrage → liste → détails ingrédients + NutriScores.
  • API documentée (Swagger) facilitant les tests depuis le navigateur.
  • Architecture claire, facilement extensible (autres sources, nouveaux champs nutritionnels).
